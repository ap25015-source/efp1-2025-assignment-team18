## 5. Δομή του κώδικα

Σύντομη περιγραφή των αρχείων:

```text
e2e_ef1-2025-assignment-team18/    # Ρίζα του project
.
├── main.py         # Είσοδος του προγράμματος / μενού / διάλογος με χρήστη
├── models.py       # Κλάσεις τομέα (π.χ. User, Request, Document, κ.λπ.)
├── services.py     # Λογική συστήματος / συναρτήσεις χειρισμού σεναρίων
└── diagrams/
    ├── class_diagram.png
    └── sequence_diagram.png
```

Περιγράψτε με 1–2 προτάσεις τι κάνει το κάθε αρχείο:

 - `main.py`:
    - περιέχει το μενού,
    - διαβάζει input από τον χρήστη,
    - καλεί τις συναρτήσεις από services.py.

- `models.py`:
    - περιέχει τις βασικές κλάσεις του συστήματος (οντότητες).

- `services.py`:
    - περιέχει συναρτήσεις που υλοποιούν τα βήματα του use case (δημιουργία/αναζήτηση/ενημέρωση αντικειμένων).

## 8. Γνωστά Περιοριστικά Σημεία / Μελλοντική Επέκταση

- Βασική επέκταση είναι η αποθήκευση των δεδομένων που δεν φαίνεται στην δεδομένη εκτέλεση του προγράμματος (δεν υπαρχει αποθηκευμένο αρχείο)
- Εξασφάλιση προσβασιμότητας και πολυγλωσσικής υπαστήριξης του προγράμματος
- Όταν ο πελάτης ζητά ένα συγκεκριμένο μέγεθος/χρώμα, ο υπάλληλος να μπορεί να ελέγχει άμεσα τη διαθεσιμότητα στο κατάστημα μέσω της φορητής του συσκευής, χωρίς να χρειάζεται να απομακρυνθεί.
